TaskFlow uses **JWT (JSON Web Token)** authentication to secure API endpoints. All protected endpoints require a valid Bearer token in the request header.

## Authentication Flow

### 1. Register a New User

Create a new user account with the `/api/auth/register` endpoint:

```json
POST /api/auth/register
{
  "username": "Joel",
  "email": "joel@test.com",
  "password": "123456"
}
```

<Info>
When a user registers, TaskFlow automatically:
- Creates a personal board for the user
- Assigns the user as the board Owner
- Creates default columns: "To Do", "In Progress", and "Done"
</Info>

Password security is handled using **SHA256 hashing** - passwords are never stored in plain text.

### 2. Login to Get a Token

Authenticate with your credentials to receive a JWT token:

```json
POST /api/auth/login
{
  "email": "joel@test.com",
  "password": "123456"
}
```

**Response:**

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

## JWT Token Structure

The JWT token generated by TaskFlow includes the following **claims**:

| Claim | Type | Description | Source |
|-------|------|-------------|--------|
| `NameIdentifier` | `ClaimTypes.NameIdentifier` | User's unique ID | `user.Id` |
| `Email` | `ClaimTypes.Email` | User's email address | `user.Email` |
| `Name` | `ClaimTypes.Name` | User's username | `user.Username` |

<Note>
These claims are extracted from the User model and embedded in the token during authentication in `AuthService.cs:134-139`.
</Note>

### Token Configuration

JWT tokens are configured in `appsettings.json`:

```json
"JwtSettings": {
  "Key": "CLAVE_SECRETA",
  "Issuer": "TaskFlowAPI",
  "Audience": "TaskFlowClient",
  "ExpiresInMinutes": 60
}
```

- **Key**: Secret key used to sign the token (HMAC SHA256)
- **Issuer**: Identifies who issued the token
- **Audience**: Identifies who can use the token
- **ExpiresInMinutes**: Token validity period (default: 60 minutes)

## Making Authenticated Requests

All protected endpoints require the JWT token in the `Authorization` header using the Bearer scheme:

```http
GET /api/board/my
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Example with cURL

```bash
curl -X GET https://api.taskflow.com/api/board/my \
  -H "Authorization: Bearer YOUR_JWT_TOKEN_HERE"
```

### Example with JavaScript Fetch

```javascript
fetch('https://api.taskflow.com/api/board/my', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
})
```

<Warning>
If you send a request without a valid token or with an expired token, you'll receive a `401 Unauthorized` response.
</Warning>

## Security Considerations

- All business endpoints require authentication
- Board membership is validated for every operation
- Role-based permissions are enforced based on the user's role in each board
- Tokens expire after the configured time period and must be refreshed
